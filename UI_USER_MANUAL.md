## 主界面布局

### 默认状态 (Prompt 模式)

这是你启动后最常看到的界面：

#### 头部区域详细说明

**Logo 区域**：
- 显示 Claude Code 的 ASCII 艺术字 logo
- 使用青色 (Cyan) 渲染
- 只在新会话开始时显示一次

**状态信息栏**：
```
┌──────────────────────────────────────────────────┐
│ ⚡ Fast Mode enabled                              │  ← 功能提示
│ 🔌 Connected to MCP: github, filesystem          │  ← MCP 连接状态
│ 💰 Cost: $0.23 (Session: $1.45)                  │  ← 用量统计
│ 📝 Session: new-project-refactor                  │  ← 会话名称
└──────────────────────────────────────────────────┘
```

可能显示的状态信息包括：
- **模型名称**：当前使用的 AI 模型
- **会话 ID**：当前会话的唯一标识
- **费用统计**：本次会话消耗的 token 和费用
- **功能开关**：Fast Mode、Auto Mode 等
- **MCP 状态**：已连接的服务
- **警告/错误**：需要用户注意的事项

#### 对话区域详细说明

这个区域占据屏幕的大部分空间，显示所有的对话历史。

**区域特性**：
- 自动滚动到最新消息
- 支持向上滚动查看历史
- 长文本自动换行
- 代码块有语法高亮
- 支持 Markdown 格式

**初始状态**：
- 如果是新会话，这个区域是空的或只有欢迎消息
- 可能显示一些快捷键提示

#### 输入区域详细说明

**固定在屏幕底部**，始终可见：

```
● _                                              ← 光标在这里闪烁
```

**组成部分**：
- **提示符** `●` ：黑色实心圆点，表示等待输入
- **输入光标**：闪烁的光标位置（默认是反色块）
- **输入区域**：可以输入文字的空间

**不同模式的提示符变化**：
| 模式 | 提示符 | 说明 |
|------|--------|------|
| 普通输入 | `●` | 等待普通文本 |
| 命令模式 | `/` | 斜杠命令 |
| Vim 模式 | `:` 或无 | Vim Normal 模式 |
| 语音录制 | `▌` + 波形 | 正在录音 |

## 消息显示区域

### 消息的基本结构

每条消息都有清晰的视觉区分：

#### 用户消息 (You)

```
┌─────────────────────────────────────────────────────┐
│ > 帮我重构 UserService 类                           │  ← 青色/绿色文字
│                                                      │
│   发送时间: 14:32:05                                │  ← 灰色小字
└─────────────────────────────────────────────────────┘
```

**特征**：
- 以 `>` 符号开头
- 用户输入的文字
- 通常用醒目的颜色（青色或绿色）
- 右侧或下方可能显示时间戳

#### AI 助手回复 (Assistant)

```
┌─────────────────────────────────────────────────────┐
│                                                      │
│ 我来帮你重构 UserService 类。让我先看一下当前的实现： │  ← 白色文字
│                                                      │
│ ┌────────────────────────────────────────────────┐   │
│ │ tool_use: ReadFile                            │   │  ← 工具调用框
│ │ file: src/services/UserService.ts             │   │
│ └────────────────────────────────────────────────┘   │
│                                                      │
│ 正在读取文件... ◐                                    │  ← 加载动画
│                                                      │
└─────────────────────────────────────────────────────┘
```

**特征**：
- 没有 `>` 前缀
- 白色或默认颜色文字
- 可能包含多个部分：
  - 文本回复
  - 工具调用
  - 代码块
  - 列表等

### 消息内容的显示格式

#### 1. 纯文本显示

```
好的，我已经理解了你的需求。接下来我会：
1. 分析现有代码结构
2. 设计新的接口
3. 实现重构方案
4. 编写测试用例
```

- 普通 Markdown 文本渲染
- 支持粗体、斜体、列表等
- 自动换行适应终端宽度

#### 2. 代码块显示

```
┌─────────────────────────────────────────────────────┐
│ // src/services/UserService.ts                      │
│                                                     │
│ export class UserService {                          │
│   constructor(private readonly db: Database) {}     │
│                                                     │
│   async getUser(id: string): Promise<User> {         │
│     return this.db.users.find(u => u.id === id);    │
│   }                                                 │
│ }                                                   │
│                                                     │
│ ← 87 lines →                                        │  ← 行数指示器
└─────────────────────────────────────────────────────┘
```

**代码块特性**：
- 有边框包围
- 显示文件路径（如果有）
- 底部显示总行数
- 如果代码很长，显示为折叠状态
- 可以点击展开查看完整代码
- 支持语法高亮（根据语言）

#### 3. 工具调用显示

当 AI 使用工具时，你会看到这样的界面：

**正在执行的工具**：
```
┌─────────────────────────────────────────────────────┐
│ ⚙️ Bash: npm test                                   │
│                                                      │
│ Running tests...                                     │
│                                                      │
│ ◐  (旋转的加载动画)                                  │
└─────────────────────────────────────────────────────┘
```

**工具执行完成**：
```
┌─────────────────────────────────────────────────────┐

│ PASS: src/UserService.test.ts                        │
```

**多种工具调用的分组显示**：

如果 AI 连续调用多个相似工具，它们会被自动分组：

```
┌─────────────────────────────────────────────────────┐
```

## 输入区域

### 输入区域的完整状态

#### 正常输入状态

```
● 帮我分析这个函数的性能问题_
                                       ↑ 光标在此处
```

**行为**：
- 你输入的文字直接显示
- 光标跟随输入移动
- 支持退格删除
- 支持方向键移动光标

#### 多行输入状态

当你按下 `Shift+Enter` 时，可以输入多行文本：

```
● 请帮我完成以下任务：
│ 1. 阅读 README.md 了解项目
│ 2. 找到性能瓶颈
│ 3. 提出优化方案_
│
│ [Enter=发送, Shift+Enter=换行]
```

**视觉提示**：
- 显示多行编辑器
- 左侧可能有竖线指示
- 底部显示操作提示

#### 输入过程中的自动补全

当你输入时，可能会看到补全建议：

```
● /he_
            ┌─────────────────┐
            │ /help    帮助   │  ← 补全菜单
            │ /health 健康检查 │
            │ /history 历史记录│
            └─────────────────┘
```

**触发条件**：
- 输入 `/` 时显示命令补全
- 输入文件路径时显示文件补全
- 输入 `@` 时显示符号补全

### 特殊输入状态

#### 语音输入模式

如果你启用了语音功能并开始录音：

```
● ▌█▎▊▋▌_
   ↑ 波形动画
```

**显示效果**：
- 提示符变成动态波形条
- 波形高度随音量变化
- 波形颜色可能变化（静音灰色，说话彩色）
- 录音结束后自动转换为文本

#### 粘贴大段文本

当你粘贴多行代码时：

```
● function calculateTotal(items: Item[]): number {
│   return items.reduce((sum, item) => {
│     return sum + item.price * item.quantity;
│   }, 0);
│ }_
│
│ [Pasted 5 lines, 127 chars]
```

**反馈**：
- 显示粘贴的内容
- 显示行数和字符数统计

#### 图像粘贴

如果你粘贴了截图：

```
● [🖼️ Image: 800x600, 124KB]_
                         ↑ 显示图像信息
```

**处理过程**：
1. 显示图像预览信息（尺寸、大小）
2. 可以添加描述文字
3. 发送后图像会上传并在对话中显示

## 三种视图模式

### 模式 1：Prompt 模式（默认）

**用途**：日常对话和操作

**界面布局**：
```
┌──────────────────────────────────────────┐
│ [Header - Logo + Status]                  │  ← 可见
├──────────────────────────────────────────┤
│                                          │
│ [最近的消息 - 约200条]                    │  ← 可见
│                                          │
│                                          │
├──────────────────────────────────────────┤
│ ● Input here                             │  ← 固定
└──────────────────────────────────────────┘
```

**特点**：
- 始终显示输入框
- 只显示最近的 ~200 条消息
- 更早的消息可以通过滚动查看（终端原生滚动）
- 适合快速交互

**何时使用**：
- 大多数情况下的默认模式
- 需要频繁输入的场景

---

### 模式 2：Transcript 模式

**进入方式**：按 `Ctrl+O`

**用途**：查看完整的对话历史

**界面布局**：
```
┌──────────────────────────────────────────┐
│ [Header - 隐藏或简化]                     │
├──────────────────────────────────────────┤
│                                          │
│ [完整历史 - 最后30条或全部]               │  ← 占满屏幕
│                                          │
│   Message #1                             │
│   Message #2                             │
│   Message #3                             │
│   ...                                    │
│   Message #N                             │
│                                          │
├──────────────────────────────────────────┤
│ Search: _                                │  ← 搜索框
│ [Ctrl+E to show all 123 previous]        │  ← 提示
└──────────────────────────────────────────┘
```

**特点**：
- 输入框隐藏，改为搜索框
- 默认显示最后 30 条消息
- 可以按 `Ctrl+E` 显示所有消息
- 消息带有编号，方便引用
- 支持内容搜索

**界面元素**：

**消息编号**：
```
[142] > 请解释这段代码的作用
[143] 这段代码实现了用户认证功能...
```

**截断提示**（如果没有显示全部）：
```
┌──────────────────────────────────────────┐
│ ──── 93 previous messages hidden ────    │  ← 分隔线
│ Press Ctrl+E to show all                 │
└──────────────────────────────────────────┘
```

**搜索功能**：
```
Search: refactor_                    ← 输入搜索词
Found 5 matches                      ← 匹配计数
  2/5 ▶                              ← 当前位置/总数

[Match 1] ...refactor the User class...    ← 高亮匹配
[Match 2] ...after refactoring...          ← 高亮匹配
```

**退出方式**：按 `Esc` 或 `Ctrl+O` 返回 Prompt 模式

---

### 模式 3：Fullscreen 全屏模式

**进入方式**：某些操作自动触发（如导出、详细查看）

**用途**：沉浸式查看或操作

**界面布局**：
```
╔══════════════════════════════════════════╗
║                                          ║
║  (使用整个终端屏幕)                       ║
║                                          ║
║  - 无 Header                             ║
║  - 无固定输入框                          ║
║  - 内容占满全屏                          ║
║                                          ║
║  ┌────────────────────────────────┐      ║
║  │ Virtual scrolling content      │      ║
║  │                                │      ║
║  │ (支持鼠标滚轮)                 │      ║
║  │ (支持鼠标选择)                 │      ║
║  │ (支持点击链接)                 │      ║
║  └────────────────────────────────┘      ║
║                                          ║
║  Press Esc or q to exit                  ║  ← 退出提示
╚══════════════════════════════════════════╝
```

**特点**：
- 使用备用屏幕缓冲区（Alternate Screen）
- 退出后完全恢复之前的终端内容
- 启用完整的鼠标支持
- 支持虚拟滚动（流畅处理大量内容）

**额外能力**：
- 🖱️ **鼠标选择文本**：拖拽选择，自动复制
- 🔗 **点击超链接**：URL 可点击打开浏览器
- 📜 **平滑滚动**：鼠标滚轮流畅滚动
- 🔍 **内联搜索**：实时搜索高亮

**"新消息"分隔线**：

如果在全屏模式时有新消息到达：

```
... 旧消息 ...

────── 3 new messages ──────     ← 分隔线

[新消息 1]
[新消息 2]
[新消息 3]
```

## 对话交互操作

### 发送消息

**基本操作**：
1. 在底部输入框输入文字
2. 按 `Enter` 发送
3. 消息立即出现在对话区域

**发送后的界面变化**：

```
发送前：
● 帮我写一个排序算法_

发送后（瞬间）：
┌──────────────────────────────────────────┐
│ > 帮我写一个排序算法                      │  ← 你的消息出现
├──────────────────────────────────────────┤
│                                          │
│ ◐ Thinking...                            │  ← AI 开始思考
│                                          │
├──────────────────────────────────────────┤
│ ● _                                      │  ← 输入框清空
└──────────────────────────────────────────┘
```

### 中断 AI 回复

当 AI 正在生成回复时，你可以中断它：

**操作**：按 `Ctrl+C` 或 `Esc`

**界面反馈**：

```
中断前：
好的，我来帮你写一个快速排序算法。快速排序的分治策略如下：
1. 选择基准值（pivot）...
2. ...

中断后：
好的，我来帮你写一个快速排序算法。快速排序的分治策略如下：
```

### OSC 8 超链接

终端支持的超链接（鼠标悬停或点击时可见）：

**链接样式**：
```
See documentation at: https://docs.claude.ai/guide
                         ^^^^^^^^^^^^^^^^^^^^^^^^^^^^
                         下划线 + 不同颜色（可点击）
```

**悬停效果**（支持的终端）：
- 链接 URL 显示在状态栏
- 光标变为手型

**点击行为**：
- 单击：延迟 500ms 后打开浏览器（等待可能的取消）
- 双击：取消打开，改为选词

### 鼠标选择与复制

**选择模式**：

1. **字符模式**（默认）：精确到每个字符
2. **单词模式**（双击后）：吸附到单词边界
3. **行模式**（三击后）：整行选择

**复制方式**：
- **自动复制**：选择完成后自动复制（取决于终端设置）
- **手动复制**：右键菜单 → Copy，或 `Cmd+C` / `Ctrl+Shift+C`

**跨平台差异**：
- macOS Terminal/iTerm2：自动复制
- Windows Terminal：需要手动复制
- Linux 终端：取决于配置

### 鼠标不工作的情况

如果鼠标操作无效，可能的原因：

1. **终端不支持鼠标**
   - 解决：使用键盘操作代替

2. **SSH 连接**
   - 解决：确保 SSH 启用了鼠标转发 (`AllowRemoteForwarding`)

3. **Tmux/Screen 会话**
   - 解决：需要在 Tmux 配置中启用鼠标 (`set -g mouse on`)

4. **远程开发环境**
   - 部分 IDE 内置终端有限制

### 颜色含义

| 颜色 | 含义 | 出现场景 |
|------|------|----------|
| 🟢 绿色 | 成功、完成、允许 | ✓ 标记、通过的消息 |
| 🔴 红色 | 错误、失败、拒绝 | ✗ 标记、错误消息 |
| 🟡 黄色 | 警告、进行中 | ⚠️ 警告、加载动画 |
| 🔵 青色/蓝色 | 信息、链接 | 用户消息、超链接 |
| ⚪ 灰色 | 次要信息、禁用 | 时间戳、禁用按钮 |
| 🟣 紫色 | 特殊标记 | AI 思考过程 |

### 图标含义

| 图标 | 含义 |
|------|------|
| `●` | 输入提示符 / 进行中（实心圆） |
| `○` | 待处理（空心圆） |
| `◐` | 进行中（半填充圆） |
| `✓` | 成功完成 |
| `✗` | 失败/错误 |
| `⚙️` | 工具执行 |
| `⚠️` | 警告 |
| `❌` | 严重错误 |
| `🔍` | 搜索 |
| `📝` | 文件写入 |
| `📂` | 文件读取 |
| `💻` | 命令执行 |
| `🔗` | 链接 |
| `📋` | 复制/剪贴板 |
| `🔊` | 声音/语音 |
| `🎤` | 录音 |
| `🖼️` | 图片 |
| `📦` | 包/依赖 |
| `🔐` | 安全/权限 |
| `💰` | 费用/成本 |
| `⚡` | 快速/性能 |
| `🤖` | 自动/AI |

### 状态指示器

| 指示器 | 含义 |
|--------|------|
| `◐ ◑ ◒ ◓ ◐` | 旋转动画（处理中） |
| `█▊▋▌█▎▊` | 波形动画（音频/语音） |
| `███████░░░░` | 进度条（百分比） |
| `▸` / `▾` | 折叠/展开（向右箭头） |
| `►` / `▼` | 播放/展开（三角形） |
| `[+]` / `[-]` | 展开/折叠（方括号） |
