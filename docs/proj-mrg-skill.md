# 项目技能需求说明


## 使用时机

要建立、跟踪需要长期执行的、基于目标导向的多任务安排、多职能Agent配合以及具有大量周期性任务的项目。

例如：
- 软件开发，需求调研/产品设计，架构设计，架构审计，代码编写，代码审计，测试，部署，持续迭代；
- 社区账号运营，根据人设持续策划新的话题，编写/发表社区文章，持续在社区各处与其它版块的用户互动（评论，点赞，关注等）；
- 数据分析，根据用户指研究标的收集数据，对数据进行整理，分析，可视化，可视化结果分享给用户；

诸如此类。

## 基本要求

需要充分与用户沟通，因为用户可能提出的只是一个简单而且模糊的想法。要引导出用户真正可被量化的目标是什么。

用户的目标可能非常之宏大，需要被分解成多个的小目标，而这个分解过程则是你非常重要的推理步骤。当全部被分解的目标都被用户确认，则可以对每个目标进行任务级别的分解，分配相关专业的Agent, 在固定时间点执行。甚至需要按特定的循环周期执行。

用户可以通过询问：“xxx 项目的进度如何”，就能查询该项目某个目标的执行情况，通过简报形式对整个项目中各个任务当前的执行请求，目标完成情况进行总结，以Markdown格式输出。

如果用户要出具具体的项目图表，则可以根据项目结构，计划时间，执行时间，状态等的信息用Mermaid画出一份项目的甘特图用Markdown格式输出给用户；



## 技术要求

1. 用 gograph 作为数据库，它是一个基于命令行的图数据库，可以直接通过cli编写 cyhper 语法来操作数据库。
2. 对于计划任务可以使用mindx的scheduler机制，用webstocket方式安排每个任务的执行时间。
3. 每个任务可以分配给不同的Agent, 每个Agent都有自己的技能和权限。
4. 每个任务的执行结果需要被记录到数据库中，用户可以通过查询数据库来查看任务的执行情况。
