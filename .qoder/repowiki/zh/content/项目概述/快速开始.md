## 简介
MindX 是一款轻量级、具备思考能力且可自主进化的 AI 个人助理。它采用仿生大脑架构，最大化发挥本地大模型能力，绝大多数日常任务可在本地完成，仅在必要时调用云端模型，从而显著降低 Token 消耗与算力成本。项目提供 Web 仪表盘与 TUI 终端两种交互方式，并支持技能生态扩展与 MCP 协议。

## 系统要求
- 操作系统：macOS / Linux / Windows
- 内存：建议 8GB 以上
- 硬盘空间：建议 20GB 以上
- 网络：首次安装需下载模型，后续可离线使用

## 安装方式
提供两种安装方式：预编译包安装与源码编译安装。

### 预编译包安装（推荐）
- 下载对应系统的发布包（GitHub Releases / Gitee Releases）
  - macOS (Universal)：同时支持 Intel 与 Apple Silicon
  - Linux (x86_64) / (ARM64)
  - Windows (x86_64) / (ARM64)
- 解压并安装
  - macOS / Linux：解压后进入目录，运行安装脚本
  - Windows：解压后运行安装脚本
- 启动服务
  - mindx start：启动后端服务
  - mindx dashboard：打开 Web 界面（默认：http://localhost:911）
  - mindx tui：启动终端界面

## 启动服务
提供多种启动方式与子命令，满足不同使用场景。

- mindx dashboard
  - 在系统默认浏览器中打开 Web 仪表盘（默认端口 911）
  - 若无图形环境或缺少 xdg-open，将打印访问地址

- mindx tui
  - 启动 TUI 终端聊天界面
  - 支持指定端口与会话 ID 参数

- mindx model test
  - 测试模型兼容性
  - 可测试全部模型或指定模型名称

- mindx kernel run
  - 运行内核服务（阻塞式）
  - 用于开发或后台运行

- mindx kernel start / stop / restart / status
  - 管理系统服务（macOS / Linux / Windows）
  - 提供状态查询与启停控制

- mindx version
  - 显示当前版本信息（包含构建时间与 Git 提交）

- mindx send
  - 向 MindX 发送消息并等待响应
  - 支持指定端口与消息内容

## 开发与调试
- 开发模式
  - make dev：启动后端与前端开发服务器（端口 911 与 5173）
  - 使用 .dev 目录作为临时工作区，避免污染用户实际工作区
  - 支持前端热重载与后端热重载

- 环境检查
  - make doctor：检查系统依赖、Ollama 模型、安装状态、工作区、权限与端口占用情况
  - 输出通过项、警告与错误统计，并给出修复建议

- 目录结构
  - 安装目录（默认：/usr/local/mindx）
  - 工作目录（默认：~/.mindx）
  - 配置文件：server.yml、models.yml、capabilities.yml、channels.yml
  - 数据目录：memory、sessions、vectors、training
  - 日志目录：logs

- 端口说明
  - Dashboard 默认端口：911
  - WebSocket 默认端口：1314
  - 可在配置文件中修改端口

## 常见问题与故障排除
- Ollama 未安装或未启动
  - 确保已正确安装并启动 Ollama 服务（验证：ollama list）
  - 可使用安装脚本自动安装与拉取模型

- 端口被占用
  - Dashboard 默认端口：911
  - WebSocket 默认端口：1314
  - 修改端口：编辑工作区配置文件中的 server.yml

- 权限问题
  - 确保工作目录具有读写权限
  - Linux/macOS：调整目录权限
  - Windows：以管理员身份运行

- Dashboard 静态文件缺失
  - 确保已执行构建（make build）
  - 开发模式使用 make dev

- 更新与卸载
  - 更新：make update（一键更新到最新版本）
  - 卸载：make uninstall 或手动删除安装目录与工作目录

- 环境检查
  - 使用 make doctor 获取详细的检查结果与修复建议
