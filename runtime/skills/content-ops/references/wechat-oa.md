# 公众号写作规范 (WeChat Official Account Writing Standard)

> 本文件是 content-ops 技能的**平台参考标准**，不是独立技能。当需要为公众号生产内容时，由 SKILL.md 的工作流按需加载本文件。

## 平台本质

公众号 = **深度内容阵地 + 私域流量入口**

2026年的公众号已进入 **"双引擎时代"**：订阅分发 + 算法推荐 + 社交裂变，三者协同运作。新号也能获得10万+阅读，前提是内容质量过关。

## 核心机制

### 三引擎分发模型

```
┌─────────────────────────────────────────────┐
│              公众号流量三引擎                 │
│                                             │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  │
│  │ 订阅引擎  │  │ 推荐引擎  │  │ 社交引擎  │  │
│  │ (粉丝)    │  │ (算法)   │  │ (裂变)   │  │
│  │           │  │           │  │           │  │
│  │ ·固定推送 │  │·信息流推荐│  │·朋友圈转发 │  │
│  │ ·星标防丢 │  │·话题标签 │  │·群聊分享  │  │
│  │ ·习惯养成 │  │·相似文章 │  │·在看激发  │  │
│  └──────────┘  └──────────┘  └──────────┘  │
│       ↑            ↑            ↑          │
│       └────────────┴────────────┘          │
│                  总阅读量                    │
└─────────────────────────────────────────────┘
```

### 用户阅读漏斗

```
打开率(标题+封面) → 完读率(开头+结构) → 分享率(价值+情绪) → 关注转化(人设+连续性)
     ↓                ↓               ↓              ↓
   前3秒决定        前300字决定      结尾决定        全文决定
   是否点击         是否继续读       是否转发        是否关注
```

**关键数据基准（2026年）：**
- 平均打开率：2-5%（优秀：8%+）
- 平均完读率：15-30%（优秀：50%+）
- 平均分享率：1-3%（优秀：8%+）
- 最佳字数区间：**800-2000字**（完读率最高）

## 一、标题工程

标题是公众号的 **第一道生死线**。决定了打开率的80%以上。

### 4U原则

| U | 含义 | 技巧 | 示例 |
|---|------|------|------|
| **Urgent** | 紧迫感 | 时间限制 / 错失恐惧 | 「2026年了，你还在用这种方法吗？」 |
| **Unique** | 独特性 | 稀缺信息 / 反共识 | 「99%的人都不知道的xx技巧」 |
| **Ultra-specific** | 超具体 | 数字化 / 场景化 | 「我用这3个方法让公众号涨粉5000」 |
| **Useful** | 有用性 | 可操作 / 可套用 | 「一份能直接套用的xx模板」 |

### 高CTR标题模式库（6种）

| 模式 | 结构 | 示例 | 心理学原理 |
|------|------|------|-----------|
| **数据+结论** | 「数字+对象+惊人结论」 | 「分析了100篇10万+，我发现了一个规律」 | 数据权威感 |
| **反常识** | 「颠覆认知的陈述」 | 「越努力越穷？你可能陷入了这种陷阱」 | 认知冲突引发好奇 |
| **身份锚定** | 「特定人群+痛点/收益」 | 「产品经理必看的5个竞品分析方法」 | 身份代入感 |
| **对话式** | 「直接对读者说话」 | 「关于涨粉，我有几句真心话想说」 | 亲密感降低防御 |
| **悬念式** | 「话只说一半」 | 「那个让我月入5位数的副业，我终于敢说了」 | 信息缺口驱动点击 |
| **权威背书** | 「来源+发现」 | 「腾讯内部培训资料流出，关于xx的思考」 | 权威转移 |

### 标题避坑

- ❌ 超过30字（微信推送通知截断 + 信息过载）
- ❌ "最新""必看""重磅"等过度营销词（用户已脱敏）
- ❌ 与正文内容不符（标题党导致取关）
- ❌ 含敏感政治/社会话题词（审核不通过）

## 二、黄金开头（前3段=生死线）

用户平均只给 **3-5秒** 判断是否继续读。开头必须完成一个任务：**制造"继续读下去"的冲动。**

### 5种高留存开头模板

#### 模板A：故事切入

```markdown
{一个具体的、有画面感的瞬间/场景}
（示例："上周二晚上11点，我对着电脑屏幕上的第17版方案，删掉了全部内容。"）

→ 这个瞬间让我意识到：{核心洞察}
```

**适用：** 个人经历类 / 观点输出类 / 创业故事类

#### 模板B：数据震撼

```markdown
{一个反直觉的数据或统计}
（示例："过去一年，我追踪了200个公众号的数据，发现一个反直觉的事实：那些涨粉最快的号，发的文章数量反而最少。"）

→ 这意味着：{数据解读}
→ 而大多数人并不知道...
```

**适用：** 行业分析 / 数据报告 / 方法论验证类**

#### 模板C：痛点共鸣

```markdown
{描述一个广泛存在的痛点场景}
（示例："你是不是也这样——每天打开公众号后台，看到阅读量只有几百，点赞是个位数，然后开始怀疑自己是不是不适合做这个？"）

→ 你是不是也遇到过？{具体症状}
→ 我花了{时间}才找到答案...
```

**适用：** 教程 / 方法论 / 自我提升类**

#### 模板D：金句暴击

```markdown
{一句有力量的判断/观点}
（示例："大多数人的勤奋，只是在用战术上的勤奋掩盖战略上的懒惰。"）

→ 但很少有人真正理解这句话的含义。
→ 今天我想聊聊...
```

**适用：** 观点 / 认知升级 / 思维模型类**

#### 模板E：承诺前置

```markdown
这篇文章会花你{时间}读完，但你将获得：

1. {收获1 — 具体可感知}
2. {收获2 — 具体可感知}
3. {收获3 — 具体可感知}

→ 如果你正在{痛点状态}，建议立刻阅读。
```

**适用：** 长文 / 深度教程 / 综合指南类**

## 三、正文结构（呼吸感编排）

公众号文章是在手机屏幕上阅读的。**密集文字 = 高跳出率。**

### 理想节奏模型

```
开头（hook）       : 100-200字  ← 制造"继续读"的冲动
                        ↓
第一层论点         : 300-500字  ← 配一张图/引用框/案例
                        ↓ [视觉断点]
过渡句             : 1-2句      ← 承上启下
                        ↓
第二层论点         : 300-500字  ← 再配一张图/表格
                        ↓ [视觉断点]
过渡句             : 1-2句      ← 节奏调节
                        ↓
第三层论点/深化    : 300-500字  ← 核心观点的最强表达
                        ↓
结尾（CTA/反思）   : 100-200字  ← 给用户行动或思考
```

### 排版铁律

- 段落不超过 **4行手机屏幕**（约80-120字）
- 每隔 **300-400字** 必须有一个视觉断点
- 重点句子 **加粗** 或用引用块突出
- **14-16px字号**，行间距1.6-1.75，段间距大于行间距
- 配色不超过 **3种**（主色 + 强调色 + 中性色）
- 两端对齐（非左对齐），手机端更整洁

### 视觉断点类型

| 类型 | 用途 | 频率 |
|------|------|------|
| 图片/截图 | 说明、佐证、调节节奏 | 每300-400字一张 |
| 引用块 | 金句、重点强调、他人观点 | 每500字一次 |
| 表格 | 对比、数据展示、清单 | 按需 |
| 加粗句 | 段落核心观点 | 每段1-2处 |
| 分隔线 | 大章节切换 | 每个H2之间 |
| 空行 | 段落间呼吸 | 必须有 |

## 四、结尾设计（分享驱动）

公众号的价值不仅在于被读完，更在于 **被传播**。结尾的设计直接决定分享率和关注转化。

### 结尾四要素

```markdown
【1. 总结收束】
用一句话回顾全文核心观点（让没耐心看完的人也能get到重点）

【2. 行动号召（CTA）】
→ 实操类："今天就可以做的第一步：{具体动作}"
→ 思考类："你最认同文中哪个观点？欢迎留言告诉我"
→ 学习类："建议收藏，下次写{相关主题}时对照着来"

【3. 社交货币】
设计一句值得转发的话（替用户想好转发理由）：
→ "转发给需要的人"
→ "转给你的老板/同事看看"
→ "发给那个也在纠结{问题}的朋友"

【4. 下期预告】（培养阅读习惯和期待感）
→ "下一篇我们聊{话题}，记得星标不错过~"
→ "这个系列还会继续，下一期讲{具体主题}"
```

## 五、分发优化（双引擎适配）

### 订阅引擎优化（服务已有粉丝）

- **固定推送时间**：培养粉丝阅读习惯（建议同一时间段±30分钟内）
- **文末引导星标**："星标防止走丢"（防止被算法淹没）
- **话题连载**：系列内容让粉丝有期待（"这是XX系列第N篇"）
- **互动回复**：发布后1小时内回复留言区（激活粉丝）

### 推荐引擎优化（获取新读者）

- **前100字包含话题关键词**：算法识别内容分类的核心信号
- **内容密度高**：算法偏好"有价值"的内容（信息量大 > 水字数多）
- **互动埋点**：文末提问 / 投票 / 留言区主动回复
- **话题标签使用**：合理利用公众号的话题标签功能
- **时长控制**：800-2000字是完读率甜区（太短不够深，太长读不完）

### 社交裂变引擎优化（放大传播）

- **设计"转发理由"**：替用户想好转发文案（"这篇文章说出了我一直想说的..."）
- **朋友圈文案预置**：方便读者一键复制到朋友圈
- **群聊适配**：设计适合在群里讨论的话题切口（"你们公司也有这个问题吗？"）
- **在看引导**：温和地提示"如果觉得有用，点个'在看'让更多人看到"

## 六、发布规范

### 最佳推送时间

| 时段 | 适合内容 | 打开率特征 |
|------|---------|-----------|
| 7:00-9:00 | 早安/新闻/行业动态 | 打开率中等，适合资讯类 |
| **12:00-13:00** | **轻松/干货/中篇文章** | **午间高峰，打开率较高** |
| **18:00-20:00** | **深度/长文/观点类** | **下班黄金时段，完读率最高** |
| 21:00-23:00 | 情感/故事/夜读 | 晚间模式，分享率高 |

### 发布后24小时运营

- **1小时内**：回复精选评论（前5-10条）
- **6小时内**：监控分享数据，如有爆转迹象及时互动
- **24小时内**：记录核心指标（打开率/完读率/分享率/新增关注）
