# 小红书写作规范 (Xiaohongshu Writing Standard)

> 本文件是 content-ops 技能的**平台参考标准**，不是独立技能。当需要为小红书生产内容时，由 SKILL.md 的工作流按需加载本文件。

## 平台本质

小红书 = **搜索引擎 + 种草社区**

用户带着明确需求来搜索，你的任务是让他在搜索结果中 **点你 → 信你 → 买你（或收藏你）**。

## 核心机制

### CES 评分（决定内容分发量的核心算法）

```
CES = 点赞×1 + 收藏×1 + 评论×4 + 转发×4 + 关注×8
```

| 行为 | 权重 | 含义 |
|------|------|------|
| 点赞 | ×1 | 轻度认可，成本最低 |
| 收藏 | ×1 | 有用，但未必认同 |
| **评论** | **×4** | 深度互动，算法认为"有讨论价值" |
| **转发** | **×4** | 社交传播，算法认为"值得推荐给更多人" |
| **关注** | **×8** | 最高价值——用户愿意持续接收你的内容 |

**策略启示：** 设计内容时优先考虑如何触发高权重行为（评论/转发/关注），而非追求点赞数。

### 流量来源结构

- **40-60% 的流量来自搜索**（SEO 是第一优先级）
- 剩余来自推荐流 + 关注流
- 素人账号贡献 **62% 的爆款** —— 不需要大V，好内容本身就能爆

### 用户心理

- "我来找答案/找方案" 不是 "我来刷内容"
- 决策场景：买什么 / 怎么做 / 避开什么坑
- 信任建立路径：封面吸引 → 标题匹配需求 → 内容解决疑虑 → 收藏备用

## 一、标题工程

标题是小红书的 **SEO 入口 + 点击决策点**。在搜索结果中，用户只看标题和封面决定是否点击。

### 7大爆款标题公式

| 公式类型 | 结构模板 | 示例 |
|---------|---------|------|
| **数字清单型** | 「数字+对象+结果」 | 「5个APP让我效率翻3倍」「3步搞定Excel透视表」 |
| **对比反差型** | 「A vs B / 之前vs之后」 | 「月薪5K和月薪50K的PPT差距」「入职前vs入职后」 |
| **避坑指南型** | 「千万别/后悔没/避雷」 | 「千万别直接卸载这个APP！」「装修10大坑，我踩了8个」 |
| **揭秘内幕型** | 「行业内幕/没人告诉你」 | 「设计师不会告诉你的3个省钱技巧」「HR筛简历的内幕」 |
| **情绪共鸣型** | 「终于/太真实了/谁懂啊」 | 「谁懂啊！搬进新家后的第100天...」「终于把阳台改造成了书房」 |
| **教程步骤型** | 「手把手/从0到1/保姆级」 | 「从0到1搭建个人知识库（保姆级教程）」 |
| **资源合集型** | 「必备/神器/清单」 | 「2026年最值得买的10件家居好物」「打工人必备的8个免费工具」 |

### 标题禁忌

- ❌ 标题党但内容空洞（"看完震惊""不转不是中国人"）—— 2026年用户免疫
- ❌ 过于文艺/抽象（"那些关于成长的碎碎念"）—— 搜索命中率低
- ❌ 全英文/日文标题 —— 降低中文搜索命中率
- ❌ 超过20字 —— 手机端显示截断

### SEO 标题技巧

- 核心搜索词放在 **前12个字内**（搜索结果显示优先级最高）
- 标题中包含用户会搜的 **具体词汇**（不用同义词替换核心词）
- 可以适当加入表情符号增加视觉吸引力（但不要滥用）

## 二、封面设计规范

封面是点击率的 **物理基础**。在小红书的信息流中，封面决定了一篇笔记 70% 以上的点击率。

### 封面四要素检查清单

```
□ 清晰度足够（手机上看得清所有文字）
□ 有字（封面必须有核心信息文字，纯图片点击率低30%+）
□ 对比（前后对比/好坏对比/数字对比制造视觉冲击）
□ 人脸（有人脸的封面点击率高15-20%，美妆/穿搭/生活方式类必用）
```

### 封面文字规范

- 主标题：**不超过8个字**，字体够大，手机上一眼能看清
- 副标题：补充说明，2-6个字
- 数字/标签：用色块或图标突出（如"第X篇""已帮XX人"）
- 配色：不超过3种，与品牌色调一致

### 封面类型选择

| 内容类型 | 推荐封面风格 | 示例 |
|---------|------------|------|
| 教程/干货 | 大字报式（关键词+数字） | "3步""5分钟""100%" |
| 好物推荐 | 产品图+价格/评分标签 | 商品图+"￥XX""9分" |
| 经验分享 | 人脸+文字叠加 | 本人照片+"我试了X天" |
| 对比测评 | 左右分屏/前后分屏 | Before→After |
| 清单/合集 | 网格式排列 | 多产品平铺+编号 |

## 三、正文写作规范

### 内容结构模板（问题-方案-行动模型）

```markdown
# {标题}

📌 【痛点开场】一句话戳中目标用户的痛点或焦虑
（示例："你是不是也经常打开电脑坐了一下午，却不知道干了什么？"）

━━━━━━━━━━━━━━━━━━

✅ 方案一：{具体方法名}
→ 操作步骤/关键要点（每步一行，简洁）
→ 适用场景/效果预期
💰 成本/代价：（如有，增加可信度）

✅ 方案二：{具体方法名}
→ ...

✅ 方案三：{具体方法名}
→ ...

━━━━━━━━━━━━━━━━━━

🎯 总结/对比表（让用户快速决策）
| 方案 | 成本 | 效果 | 适合人群 |
| A | 低 | ⭐⭐⭐ | 新手 |
| B | 中 | ⭐⭐⭐⭐ | 进阶 |

💬 互动引导：
→ "你用的是哪个？评论区告诉我～"
→ "还有其他方法吗？欢迎补充！"
→ "收藏起来慢慢看 👇"

#小红书 #{精准标签3-5个} #{长尾标签2-3个}
```

### 正文写作铁律

**分段要短**
- 手机屏幕宽度，每段 **2-3行最多**
- 大量留白，让用户可以快速扫读
- 一段只讲一个观点

**emoji 当标点**
- 用 emoji 替代项目符号，增加视觉节奏感
- 但不要过度使用（一篇笔记 emoji 数量控制在 10-15 个以内）
- 常用：✅ 📌 🎯 💡 💰 🔥 ⭐ 💬 ➡️ 📝

**口语化表达**
- 像跟闺蜜聊天，不要像写论文
- 合理使用口语词（"咱就是说""真的绝了""家人们""谁懂啊"），但要自然不刻意
- 避免书面语和学术表达

**具体 > 抽象 > 空洞**
- 不说"提高效率"，说"每天省出2小时"
- 不说"很便宜"，说"9块9包邮"
- 不说"很多人都在用"，说"我已经用了3个月"
- 不说"效果显著"，说"我的数据从500涨到了8000"

**收藏价值 = 爆款潜力**
- 写完问自己：**"用户会截图保存吗？"**
- 工具类内容：步骤完整、可直接操作
- 干货类内容：信息密度高、有独特见解
- 清单类内容：全面、可检索、可回看

## 四、SEO 关键词布局

小红书 40-60% 的流量来自搜索，SEO 不是可选项而是必修课。

```
┌─────────────────────────────────────────────┐
│  标题：必须包含核心搜索词（放在前12个字内）    │
│  ↓                                          │
│  正文前两段：自然融入 2-3 个相关搜索词         │
│  ↓                                          │
│  正文中间：长尾词自然出现（不要堆砌）          │
│  ↓                                          │
│  标签：{核心词} + {相关词} + {长尾词}        │
│  ↓                                          │
│  图片Alt/OCR文字：覆盖更多搜索场景            │
└─────────────────────────────────────────────┘
```

### 关键词研究方法

1. **搜索框下拉建议**：输入核心词，查看小红书推荐的联想词
2. **竞品分析**：查看同领域爆款笔记用了哪些关键词
3. **"大家都在搜"**：小红书搜索页底部的热门搜索词
4. **第三方工具**：灰豚数据、千瓜数据、新红等（如有）

### 标签策略

```
标签结构 = {核心大词 1-2个} + {精准中词 3-5个} + {长尾词 2-3个}

示例（家居收纳主题）：
#收纳 #家居收纳 #小户型收纳   ← 核心/中词
#租房改造 #衣柜整理法 #10元好物  ← 长尾词
```

## 五、互动设计（CES 优化）

提升 CES 权重不是靠运气，而是靠 **设计**。

| 互动类型 | 触发方式 | 具体技巧 | CES权重 |
|---------|---------|---------|---------|
| 评论（×4） | 正文留白/故意不完整/提问/争议点 | "你遇到过吗？""还有第N种方法，有人知道吗？" | 高 |
| 转发（×4） | 极高实用价值/情绪共鸣强/替用户说话 | 完整方法论 / 说出了用户想说的话 | 高 |
| 关注（×8） | 人设鲜明/持续输出同类价值/预告后续 | "下期分享XX""关注我不迷路" | 最高 |
| 收藏（×1） | 工具属性强/步骤完整/值得回看 | 清单/模板/教程/对比表 | 中 |
| 点赞（×1） | 认同感/美观/有用 | 认同表态/好看的内容 | 低 |

### 高评论率设计技巧

1. **故意留白**："其实还有第4种方法，但是..."（引导追问）
2. **投票式提问**："A方案还是B方案？评论区告诉我你的选择"
3. **求助式**："我也在想XXX，有没有姐妹有更好的办法？"
4. **争议点预设**：主动提出一个有争议的观点，引发讨论
5. **@好友**：在正文中提及"@某某 之前问我"，引导她来评论

## 六、发布规范

### 最佳发布时间（女性用户为主）

| 时段 | 适合内容类型 | 原因 |
|------|------------|------|
| 7:00-9:00 | 早安/效率/正能量 | 通勤时间刷手机 |
| 11:30-13:00 | 好物/美食/轻松话题 | 午休放松 |
| 17:30-19:00 | 家居/穿搭/生活 | 下班路上规划 |
| **20:00-23:00** | **所有类型（黄金时段）** | **活跃度最高，发完立刻有互动** |
| 0:00-2:00 | 情感/深夜话题 | 夜猫子用户 |

### 发布后运营（24小时内）

- **发布后30分钟**：回复前3条评论（激活初始互动）
- **发布后2小时**：回复精选评论，自己也可以补充评论补充信息
- **发布后12小时**：二次回复，检查是否有负面评论需处理
- **发布后24小时**：记录数据表现，用于下周复盘
