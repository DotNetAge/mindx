# 抖音脚本规范 (Douyin Script Writing Standard)

> 本文件是 content-ops 技能的**平台参考标准**，不是独立技能。当需要为抖音生产短视频内容时，由 SKILL.md 的工作流按需加载本文件。

## 平台本质

抖音 = **注意力竞技场**

你只有 **3秒** 证明自己值得看。用户来这里是为了 **杀时间**，不是来学习的 —— 所以内容必须是 **娱乐外壳 + 价值内核**。

## 核心机制

### 3秒法则

```
前3秒 → 决定80%的完播率
前5秒 → 决定60%的用户去留
前15秒 → 决定是否进入推荐池的大流量分发
```

**如果前3秒不能让拇指停止滑动，后面写得再好也没人看。**

### 完播率是一切的基础

```
完播率低 → 算法判定内容质量差 → 不给推 → 播放量死
完播率高 → 算法判定内容优质 → 进入更大推荐池 → 播放量爆发
```

### 黄金前5秒 = 三位一体

| 维度 | 要求 | 失败后果 |
|------|------|---------|
| **视觉冲击** | 最抓眼球的画面/表情/动作/对比 | 用户不看 |
| **听觉钩子** | 最有杀伤力的台词/BGM/音效 | 用户不听 |
| **信息悬念** | 最让人好奇的问题/冲突/结论 | 用户不关心 |

**三者缺一不可。** 只有画面没有钩子 = 美但没有理由留下；只有钩子没有画面 = 有趣但不够刺激。

## 一、脚本结构模板

### 标准短视频脚本（30-90秒）

```markdown
## 抖音脚本 — [{预计时长}秒] — [{内容类型}]

═════════════════════════════════════════════
🎯 核心信息（一句话）：{用户看完后记住的一句话}

🎯 目标完播率：{X%}  |  🎯 目标互动率：{X%}
═════════════════════════════════════════════

【0-3秒】黄金钩子（HOOK）
画面：{最冲击的画面/表情/动作/对比}
台词："{最有杀伤力的第一句话}"
字幕：{大字突出核心冲突/悬念}
音效：{BGM起/特殊音效/突然静音}
→ 目标：让拇指停止滑动

【3-15秒】快速建立信任/共情
画面：{讲述者出镜/演示过程/场景切换}
台词："{我是谁 / 你是否有这个问题 / 结果有多严重}"
→ 目标：让用户觉得"这说的就是我"

【15-{N-15}秒】干货输出 / 情绪推进
画面：{分步演示 / 案例 / 对比 / 数据展示}
台词："{核心方法/观点/反转/笑点}"
字幕：{关键信息高亮}
→ 目标：持续提供"不看亏了"的感觉

【{N-15}-{N-5}秒】高潮/反转/结果
画面：{最终效果/前后对比/情绪爆发}
台词："{最强记忆点 / call back / 反转结论}"
→ 目标：创造"哇"的时刻

【{N-5}-{N}秒]CTA + 引导互动
画面：{指向关注按钮/展示评论区/产品展示}
台词："{关注我看更多 / 评论区告诉我 / 左下角下单}"
→ 目标：转化动作

═════════════════════════════════════════════
拍摄备注：{景别/运镜/道具/灯光注意事项}
剪辑备注：{转场方式/BGM节点/字幕风格/节奏卡点}
```

## 二、高频Hook公式库

Hook = 前3秒让观众停止滑动的武器。

| Hook类型 | 公式 | 示例 | 适用场景 |
|---------|------|------|---------|
| **结果前置** | "做完这件事后，{惊人的结果}" | "做完这个调整后，我的效率翻了3倍" | 效果展示/前后对比 |
| **反直觉** | "你以为{常识}？其实{真相}" | "你以为早起好？其实睡够了才是关键" | 颠覆认知/知识科普 |
| **挑战权威** | "{权威说法}全是错的" | "所谓的时间管理，90%的方法都是骗人的" | 观点输出/方法论证 |
| **数字冲击** | "{数字}{单位}{惊人事实}" | "90%的人每天都在犯这个错" | 数据驱动/警示类 |
| **身份召唤** | "{特定人群}注意！" | "月薪低于1万的注意了！这个技能帮你翻盘" | 特定人群/痛点切入 |
| **悬念式** | "直到{事件}，我才明白..." | "直到被裁员那天，我才明白..." | 故事叙述/经历分享 |
| **对比式** | "{普通人} vs {高手}" | "普通人和高手之间，就差这一个习惯" | 方法对比/进阶指导 |
| **警告式** | "千万别...否则..." | "千万别直接点删除！你可能丢了重要文件" | 避坑指南/操作提醒 |

### Hook设计铁律

- **一句话说完**：Hook 不能超过一行字幕
- **具体不抽象**：不说"有个好消息"，说"我昨天涨了5000粉"
- **视觉配合**：Hook 台词出现的同时，画面必须有对应冲击
- **不要自嗨**：你觉得有趣的 ≠ 用户觉得有趣的（用数据验证）

## 三、内容类型与脚本策略

| 类型 | 最佳时长 | 脚本重点 | 互动设计 | CTA |
|------|---------|---------|---------|-----|
| **知识口播** | 45-90s | 3秒抛问题 → 快速给答案 → 方法拆解 → 总结金句 | "你觉得呢？评论区见" | "关注看更多" |
| **剧情/段子** | 30-60s | 冲突前置 → 反转 → 再反转 → 回应现实 | "猜到结局算我输" | "点赞催更" |
| **测评/开箱** | 60-120s | 先给结论 → 逐一验证 → 最终评分 | "你给打几分？" | "左下角同款" |
| **教程/演示** | 60-180s | 效果前置 → 慢速演示 → 关键细节放大 | "学会了扣1" | "收藏慢慢看" |
| **Vlog/日常** | 30-60s | 氛围感开场 → 日常片段精选 → 情感收尾 | "这样的日子你想不想拥有" | "关注一起变好" |

### 各类型关键差异

**知识口播：** 信息密度要高，每10秒至少一个"知识点"。语速偏快（220-250字/分钟），用快剪辑保持节奏。

**剧情/段子：** 前3秒必须有冲突或悬念。每个反转点控制在10-15秒内。结尾要有 call back（呼应开头）。

**测评/开箱：** 先给结论（好评/差评/中立），再展开原因。缺点也要说（增加可信度）。最终给出明确评分。

**教程/演示：** 先展示最终效果（"这是做完之后的样子"），再教步骤。关键步骤放慢 + 特写 + 重复。

**Vlog/日常：** 氛围感第一，不需要强叙事。音乐选曲占成功因素的50%。文案要像"内心独白"而非"解说词"。

## 四、节奏把控

### 信息密度标准

```
理想状态：每10秒至少一个"不看亏了"的点

这些"点"包括：
· 新信息/新知识点
· 新画面/新场景切换
· 情绪转折/笑点/泪点
· 强调/重复的关键信息
· 字幕变化/特效/音效
```

### 节奏杀手（避免）

- 同一景别超过 **8秒** 不切（观众疲劳）
- 同一段话超过 **15秒** 不打断（注意力流失）
- 没有 BGM 变化/音效点缀超过 **20秒**（听觉单调）
- 画面静止无变化超过 **5秒**（用户以为卡了）

### 节奏加速技巧

- **快剪**：每1-3秒一切（适用于高能量段落）
- **跳剪**：去掉语气词和停顿（适用于口播）
- **画外音+空镜**：说话时不一定非要露脸
- **倍速播放**：非核心段落可以1.2x-1.5x
- **音效点缀**：转场/强调/笑点处加音效

## 五、CTA设计（转化闭环）

视频的最后5-10秒是 **唯一确定的转化机会**（因为完播到这里的人是最精准的受众）。

### CTA类型矩阵

| 目标 | CTA话术 | 画面配合 |
|------|--------|---------|
| 涨粉 | "关注我，下期分享XX" | 手指指向关注按钮 |
| 互动 | "XX的在评论区扣1" | 展示评论区界面 |
| 导流 | "完整版在我的主页/公众号" | 展示主页/二维码 |
| 转化 | "左下角/橱窗有同款" | 手指指向购物车位置 |
| 分享 | "转发给需要的人" | 分享按钮动画 |

### CTA设计原则

- **只有一个目标**：不要同时求关注+评论+购买（贪多嚼不烂）
- **具体明确**：不说"支持下我"，说"点个关注"
- **降低门槛**：评论引导用"扣1"而不是"说说你的看法"
- **利益前置**：告诉用户关注后能得到什么（"关注我看更多XX技巧"）

## 六、发布规范

### 最佳发布时间

| 时段 | 适合内容 | 原因 |
|------|---------|------|
| 7:00-9:00 | 正能量/新闻/早间资讯 | 通勤第一波 |
| 11:30-13:00 | 轻松/美食/生活 | 午饭休闲 |
| 17:00-19:00 | 娱乐/搞笑/解压 | 下班放松 |
| **20:00-22:00** | **全类型（黄金时段）** | **活跃峰值，算法分发最大** |
| 22:00-0:00 | 情感/深夜/故事 | 夜猫子群体 |

### 发布后2小时黄金运营期

- **发布后15分钟**：回复前3条评论（启动初始互动）
- **发布后1小时**：回复热门评论，自己可以补充一条评论引导方向
- **发布后2小时**：检查完播率和互动数据，判断是否需要投DOU+
