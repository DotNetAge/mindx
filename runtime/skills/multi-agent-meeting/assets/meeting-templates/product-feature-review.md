# Product Feature Review Meeting Template

## Meeting Type
Product decision meeting — feature priority review

## Meeting Goal
Review product feature priorities, decide the next quarter's development plan, and ensure resources are allocated to the most valuable features.

## Recommended Participants

### Required
1. **Product manager**
   - Role: Lead discussion, present prioritization, coordinate debate
   - Focus: User value, product strategy

2. **Architect**
   - Role: Assess technical feasibility and implementation difficulty
   - Focus: Technical risk, architecture impact, development cost

3. **Market analyst**
   - Role: Assess market value and competitive landscape
   - Focus: Market demand, competitive comparison

### Optional
4. **Financial advisor**
   - Role: Assess cost-benefit and ROI
   - Focus: Input-output ratio, resource efficiency

5. **Project manager**
   - Role: Assess resources and timeline
   - Focus: Team capacity, delivery timeline, dependencies

**Chair**: Executive assistant

## Meeting Process

### Phase 1: Data Preparation (pre-meeting)
Product manager prepares a feature list including:
- Feature name and description
- Estimated user value
- Initial priority assessment

### Phase 2: Opening Statement (5 min)
1. Product manager introduces meeting goal and context
2. Present prioritization results
3. Explain evaluation criteria and decision rationale

### Phase 3: Per-Feature Review (20 min)
For each feature:
1. **Product manager**: Explain feature value and priority score
2. **Architect**: Assess implementation difficulty and technical risk
3. **Market analyst**: Assess market demand and competitive advantage
4. **Financial advisor** (if present): Assess ROI and resource efficiency

### Phase 4: Disagreement Discussion (10 min)
For features with significant score variance:
- Each agent provides detailed arguments
- Discuss whether to adjust priorities
- Identify hidden risks and opportunities

### Phase 5: Consensus Convergence (5 min)
1. Summarize all viewpoints
2. Finalize priority order
3. Confirm Q2/Q3 development plan

### Phase 6: Decision Output
Generate a meeting record including:
- Final priority order (with rationale)
- Evaluation rationale for each feature
- Technical risks and considerations
- Resource allocation and timeline
- Next action items

## Output Format

```markdown
# Product Feature Priority Review Meeting Record

**Meeting Time**: [auto-generated]
**Participating Agents**: Product manager, Architect, Market analyst, Financial advisor
**Meeting Goal**: Review Q2 feature priorities

## Final Priority Order

| Rank | Feature                | Priority | User Value | Difficulty | Market Value |
| ---- | ---------------------- | -------- | ---------- | ---------- | ------------ |
| 1    | AI auto-generation     | P0       | High       | Medium     | High         |
| 2    | Multi-platform publish | P1       | High       | Medium     | Medium       |
| ...  | ...                    | ...      | ...        | ...        | ...          |

## Decision Rationale

### AI Auto-generation (Priority: P0)
- **Product manager**: Core value feature, strong user demand
- **Architect**: Technically feasible, can leverage existing capabilities
- **Market analyst**: Competitors haven't implemented this yet, first-mover advantage
- **Financial advisor**: High expected ROI

### Multi-platform Publish (Priority: P1)
- **Product manager**: Most requested feature from users
- **Architect**: Medium implementation difficulty
- **Market analyst**: Expands user coverage

## Risks and Considerations

### Technical Risks
- New technology needs validation, may affect development timeline

### Market Risks
- Competitors may follow quickly

## Next Actions

- [ ] Product manager: Write PRD for P0 features (Due: Week X)
- [ ] Architect: Design technical solution (Due: Week X)
- [ ] Project manager: Create development plan (Due: Week X)
```

## Notes

1. **Data preparation is important**: Feature list and scores must be ready before the meeting
2. **Keep discussion focused**: Avoid diving into technical details, stay focused on priority decisions
3. **Record disagreements**: For contentious features, document each party's arguments in detail
4. **Define next steps**: Every meeting must end with clear action items and owners
