# Customer Insight Analysis Meeting Template

## Meeting Type
Product research meeting — customer insight analysis

## Meeting Goal
Analyze customer interview transcripts, extract key insights, generate product improvement proposals and PRD drafts.

## Recommended Participants

1. **Product manager** (lead presenter)
   - Role: Present findings, interpret insights, define improvement plans

2. **Market analyst**
   - Role: Analyze user feedback from a market perspective, identify trends

3. **Architect**
   - Role: Assess technical feasibility of proposed solutions

**Chair**: Executive assistant

## Meeting Process

### Phase 1: Data Preparation (pre-meeting)
Product manager prepares interview transcript summaries:
- Direct user quotes
- Core pain points
- Feature expectations

### Phase 2: Insight Presentation (product manager)
Product manager presents to the meeting:
1. Interviewee background
2. Key findings
3. User pain points and expectations

### Phase 3: Deep Discussion
1. **Product manager**: Interpret insights from a product strategy perspective
2. **Market analyst**: Assess how representative the user feedback is from a market perspective
3. **Architect**: Evaluate technical feasibility

### Phase 4: Solution Generation
1. Identify the top 3 highest-priority pain points
2. Design solutions for each pain point
3. Assess value and difficulty of each solution

### Phase 5: PRD Draft
Product manager drafts a requirements document using the PRD template

## Notes

1. **Keep interview transcripts complete**: Include direct quotes for key references
2. **Focus on user behavior**: Analyze what users actually do, not what they say they want
3. **Dig for deeper needs**: Use "why" to uncover real user needs
4. **Validate solutions**: After generating solutions, continue interviewing to validate
5. **Iterate quickly**: PRD drafts should be generated rapidly and refined through iteration
