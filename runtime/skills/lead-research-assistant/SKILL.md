# 线索研究

此技能分析你的产品/服务，理解理想客户画像，提供可操作的拓展策略，帮你识别和筛选潜在销售线索。

## 何时使用

- 为产品/服务寻找潜在客户或合作方
- 建立待联络的合作公司清单
- 识别销售拓展的目标客户
- 调研符合理想客户画像的公司
- 为商务拓展活动做准备

## 功能说明

1. **理解你的业务**：分析产品/服务、价值主张和目标市场
2. **识别目标公司**：基于以下条件寻找符合理想客户画像的公司：
   - 行业和领域
   - 公司规模和所在地
   - 技术栈和使用的工具
   - 发展阶段和融资情况
   - 你的产品能解决的痛点
3. **线索优先级排序**：根据匹配度和相关性对公司排名
4. **提供联络策略**：建议如何用个性化信息接触每个线索
5. **丰富数据**：收集决策者和公司背景的相关信息

## 使用方法

### 基础用法

描述你的产品/服务和目标：

```
我正在开发 [产品描述]。帮我找 10 家 [地点/行业] 的公司，
它们应该是这个产品的良好线索。
```

### 结合代码库使用

在产品源代码目录下运行，效果更好：

```
看看我在这个仓库里正在构建什么，识别 [地点/行业] 中
最可能从这个产品受益的前 10 家公司。
```

### 高级用法

更精准的研究：

```
我的产品：[描述]
理想客户画像：
- 行业：[行业]
- 公司规模：[规模范围]
- 地点：[地点]
- 当前痛点：[痛点]
- 使用的技术：[技术栈]

帮我找 20 个合格线索，并为每个提供联络策略。
```

## 操作指引

收到线索研究请求时：

1. **理解产品/服务**
   - 在代码目录下时，分析代码库理解产品
   - 询问价值主张的相关问题
   - 识别关键特性和优势
   - 理解它解决了什么问题

2. **定义理想客户画像**
   - 确定目标行业和领域
   - 识别公司规模范围
   - 考虑地理偏好
   - 理解相关痛点
   - 注意技术要求

3. **调研和识别线索**
   - 搜索符合标准的公司
   - 寻找需求信号（招聘广告、技术栈、近期新闻）
   - 考虑增长指标（融资、扩张、招聘）
   - 识别拥有互补产品/服务的公司
   - 检查预算指标

4. **优先级排序和评分**
   - 为每个线索创建匹配度评分（1-10）
   - 考虑以下因素：
     - 与理想客户画像的契合度
     - 即时需求信号
     - 预算可用性
     - 竞争格局
     - 时机指标

5. **提供可操作的输出**
   
   为每个线索提供：
   - **公司名称**和网站
   - **为何匹配**：基于其业务的具体原因
   - **优先级评分**：1-10 分并附说明
   - **决策者**：目标角色/职位（例如"工程副总裁"）
   - **联络策略**：个性化接触建议
   - **价值主张**：你的产品如何解决他们的具体问题
   - **对话切入点**：接触中要提及的具体要点
   - **LinkedIn 链接**：如有，便于建立联系

6. **格式化输出**

   以清晰、易扫描的格式呈现结果：

   ```markdown
   # 线索研究结果
   
   ## 摘要
   - 找到的总线索数：[X]
   - 高优先级（8-10）：[X]
   - 中优先级（5-7）：[X]
   - 平均匹配度评分：[X]
   
   ---
   
   ## 线索 1：[公司名称]
   
   **网站**：[URL]
   **优先级评分**：[X/10]
   **行业**：[行业]
   **规模**：[员工数/营收范围]
   
   **为何匹配**：
   [基于其业务的 2-3 个具体原因]
   
   **目标决策者**：[角色/职位]
   **LinkedIn**：[URL（如有）]
   
   **对他们的价值主张**：
   [对这家公司的具体收益]
   
   **拓展策略**：
   [个性化方法 - 提及具体痛点、近期公司新闻或相关背景]
   
   **对话切入点**：
   - [具体要点 1]
   - [具体要点 2]
   
   ---
   
   [对每个线索重复]
   ```

7. **提供后续步骤建议**
   - 建议将结果保存为 CSV 以便导入 CRM
   - 提供起草个性化拓展信息的服务
   - 根据时机推荐优先级排序
   - 建议对顶级线索进行后续研究

## 示例

### 示例 1：来自 Lenny's Newsletter

**用户**："我正在开发一个工具，用于在 AI 编程助手查询中遮蔽敏感数据。帮我找潜在线索。"

**输出**：创建按优先级排序的公司列表，这些公司：
- 使用 AI 编程助手（Copilot、Cursor 等）
- 处理敏感数据（金融科技、医疗保健、法律）
- 在 GitHub 仓库中有使用编程代理的证据
- 可能在代码中意外暴露了敏感数据
- 包含相关决策者的 LinkedIn 链接

### 示例 2：本地业务

**用户**："我经营一家远程团队生产力咨询公司。帮我找 10 家最近转为远程办公的湾区公司。"

**输出**：识别以下公司：
- 最近发布了远程职位
- 宣布了远程优先政策
- 正在招聘分布式团队
- 显示出远程工作挑战的迹象
- 为每家公司提供个性化拓展策略

## 最佳实践

- **具体描述**产品及其独特价值
- **在代码库中运行**（如适用）自动获取上下文
- **提供上下文**说明理想客户画像
- **指定约束条件**如行业、地点或公司规模
- **请求后续研究**获取有前景线索的更深入洞察

## 相关场景

- 识别线索后起草个性化拓展邮件
- 构建可导入 CRM 的合格潜在客户 CSV
- 详细研究特定公司
- 分析竞争对手的客户群
- 识别合作机会
