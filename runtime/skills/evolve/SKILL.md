# Evolve

Distill repeated multi-step workflows from session history into reusable skills (SKILL.md). The output is a new skill under `{skill_dir}/evolved/<name>/SKILL.md` that future sessions can match and execute.

Skills directory `{skill_dir}` is provided in system prompt.

## Prerequisite

The daemon must be running:

```bash
mindx start
```

---

## Phase 1: List Sessions

```bash
mindx session list
```

Output shows available sessions with ID, agent, and details.

Select sessions to analyze:
- Default: 3-5 most recent sessions with meaningful conversation length
- If user specifies a session ID, focus on that one
- Skip sessions with only 1-2 messages (insufficient data)

---

## Phase 2: Read Session Content

### Overview (all messages)

```bash
mindx session get --id <session-id>
```

This returns the full session with all messages in JSON format — the LLM can analyze the content directly.

---

## Phase 3: Pattern Analysis

Use your own reasoning. Look for:

1. **Recurring workflows** — multi-step action sequences appearing 2+ times with a clear trigger scenario. Minimum 3 steps in a consistent order.

2. **Capability gaps** — user requests that repeat but none of your current skills cover.

Do NOT extract generic one-off actions or trivial patterns. Be conservative.

---

## Phase 4: Generate Skills

For each identified workflow:

### 4.1 Create SKILL.md via `write` tool

Path: `{skill_dir}/evolved/<workflow-name>/SKILL.md`

```markdown
---
name: evolved-<workflow-name>
description: >
  [Auto-evolved] One-sentence trigger description.
  Include user intents that should match this skill.
allowed-tools: [tools actually used in this workflow]
---

# Evolved: <Workflow Name>

Auto-generated by evolve on YYYY-MM-DD from sessions: [ids]

## Trigger
When does this activate?

## Steps
1. Step 1
2. Step 2
```

**Rules:**
- Name: `evolved-<name>`, lowercase hyphen-separated. Prefix `evolved-` distinguishes auto-generated from hand-written skills.
- Only list tools actually used in the workflow.
- One workflow = one skill. Don't merge unrelated patterns.
- Future-you must be able to execute this without additional context.

---

## Phase 5: Report

```
Evolution complete — N new skill(s):
  evolved-xxx — description (based on sessions: a1b2, c3d4)
  evolved-yyy — description (based on sessions: e5f6)

Tip: Run `introspect` to check whether these new skills match your role.
```

If no patterns found:
```
No significant patterns detected in analyzed sessions.
```
