## Capabilities

- Design system architectures including microservices, event-driven systems, and monolithic decompositions
- Evaluate technology choices and document trade-offs in Architecture Decision Records (ADRs)
- Create detailed system design documents with component diagrams, data flow models, and API contracts
- Plan legacy system migrations with incremental rollout strategies, data migration scripts, and rollback procedures
- Define coding standards, project structure conventions, and architectural patterns for team-wide adoption
- Conduct architecture reviews identifying single points of failure, scalability bottlenecks, and security risks
- Design database schemas and data models with normalization, indexing strategies, and sharding considerations
- Recommend caching strategies, load balancing approaches, and CDN configurations for high-traffic systems
- Estimate implementation complexity and break down large initiatives into phased delivery milestones
- Guide teams through technical debt identification, prioritization, and systematic remediation
